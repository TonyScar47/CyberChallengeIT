3)Web 03 - HTTP: richiesta GET con header manuale
->curl -H "X-Password: admin" indirizzoWeb

      [-H "Nome-Header: Valore"]

[-H indica header. Serve a passare al server un'intestazione specifica che il tuo browser o
curl normalmente non invierebbero, o che vuoi modificare manualmente]

5)Web 05 - HTTP: cookie manuale

7)Web 07 - HTTP: il metodo HEAD

[La flag -I istruisce curl a fare proprio una richiesta head]

8)Web 08 - HTTP: una richiesta post tradizionale
->curl -X POST indirizzoWeb \
  -d "username=admin" \
  -d "password=admin"

{ALTERNATIVAMENTE:}
->curl -d "username=admin&password=admin" indirizzoWeb

[-X POST: specifica esplicitamente il metodo HTTP da usare
 -d: questo flag invia i dati nel corpo della richiesta. Che di default, curl aggiunge
 l'header "Content-Type: application/x-www-form-urlencoded
 -d ]

 [Nel secondo caso, scrivo direttamente "-d" siccome in automatico innesca tre comportamenti:
 1. Cambio metodo: imposta il metodo HTTP a POST;
 2. Header Content-Type: viene aggiunto automaticamente l'header;
 3. Corpo della richiesta: dopo "-d" viene inserito nel body del pacchetto HTTP]

 9)Web 09 - HTTP: una richiesta POST con body JSON
 ->curl -X POST indirizzoWeb \
   -H "Content-Type: application/json" \
   -d '{"username": "admin", "password": "admin"}'

[-H "Content-Type: application/json": Senza questo header, il server potrebbe provare a
 interpretare le parentesi graffe come se fossero una stringa;
 -d '...': uso i singoli apici all'esterno e i doppi apici all'interno perche' JSON richiede
 rigorosamente i doppi apici per le chiavi e per le stringhe]

12)Web 12 - Tecnologie web: estrarre semplici contenuti da una pagina web
->curl -s http://web-12.challs.olicyber.it/ | grep -oE "flag\{.*\}"

  [Praticamente | dice che prende l'output di cio che sta prima e lo trasforma in input per la seconda parte; grep è
  il filtro: -o sarebbe Only-Matching e -E sarebbe Extended Regexp che ti permette di usare i caratteri speciali.
  Poi "flag" dice che deve cercare testuale, poi .* praticamente prende tutto da li in poi fino a quando non trova
  la prentesi graffa di chiusura \}]
